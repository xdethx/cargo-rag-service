# 2026-06-27 — Milestone 3: Deployment (live on Hugging Face)

## What was built
Deployed the service to a **Hugging Face Docker Space** — live at
https://dethx-ragtest.hf.space. The cloud LLM runs via the reused `openai_compatible`
provider path pointed at **Groq**; embeddings stay **local** (the same
sentence-transformers model used at ingest, baked into the image). `POST /ask` is protected
by a shared-secret API key and per-IP rate limiting; `GET /health` is a cheap public
liveness check that also reports the effective provider/model.

## Key decisions & why

**HF Spaces over Render.** This followed directly from the design choice to keep embeddings
local: the sentence-transformers model needs more RAM than Render's free tier (512 MB),
while a free HF Space gives 16 GB. The "embeddings are local and free" decision drove the
hosting choice — not the other way around.

**Groq with zero code change.** Selecting Groq was pure configuration via the generic
OpenAI-compatible provider (base_url + model + key from env) — config, not code. No new
provider branch was added.

**Shared-secret API key over JWT.** No end-user identity is needed here; the .NET app calls
the service with a single shared secret. JWT would add key infrastructure for no benefit.

**Per-IP in-memory rate limit, but the API key is the real security boundary.** The rate
limit guards against runaway callers; the key guards against unauthorized ones. An attacker
without the key never reaches the limiter.

## Two things learned live (interview anecdotes)

1. **Groq deprecated the chosen model.** `llama-3.1-8b-instant` was deprecated (2026-06-17).
   The fix was a single environment variable change — no code, no redeploy of logic — which
   is exactly the payoff of the provider-agnostic design.

2. **`/health` reporting the effective provider/model caught a near-miss.** I had flipped the
   `OPENAI_COMPAT_*` vars but left `LLM_PROVIDER=ollama`, so I was about to deploy having
   tested Ollama while believing I'd tested Groq. Because `/health` echoes the *effective*
   provider/model, the mismatch was obvious before it shipped.

## Deferred for scale
- **Switch `OPENAI_COMPAT_MODEL` to the current `openai/gpt-oss-20b`** — the model currently
  in use is deprecated and only works during Groq's grace period.
- **Add a root `/` route** so the landing URL isn't a 404.
- **Auto-deploy from GitHub to HF** via a GitHub Action (currently a manual push).
- **Keep-warm pings** — HF free Spaces sleep after 48h of inactivity.
- **Redis-backed shared rate limiter** once there's more than one instance (in-memory
  counters are per-process and don't enforce a global limit).
- **Managed vector DB (pgvector / Qdrant)** when the corpus grows or needs metadata filtering.
